# 注塑车间排程 — 工厂建模方案（简化版）

面向注塑行业车间排程场景，采用 **车间 → 注塑机（Resource）** 两级结构，增强 Resource 实体的注塑机属性，关联工厂日历。

---

## 注塑行业排程特点

- **核心设备**：注塑机，按锁模力（吨位）分组，决定可生产的模具范围
- **排程粒度**：单台注塑机 = 一个排程资源
- **关键约束**：注塑机吨位 ≥ 模具所需吨位、换模时间
- **组织结构简单**：通常单工厂、单/少量车间，不需要产线/设备等多层嵌套

## 现状 → 目标

| 现状 | 目标 |
|------|------|
| `Resource` 只有 code/name/type/available | 增加注塑机属性（吨位、机台状态、所属车间、日历） |
| 无车间概念 | 新增 `Workshop` 作为注塑机的分组和日历挂载点 |
| `FactoryCalendar` 全局 | 车间可关联特定日历 |

## 数据模型

### 层级关系（只有两级）

```
Workshop (车间)  ← 关联日历
  └── Resource (注塑机)  ← 已有表，增强字段
```

### 1. 新增 Workshop（车间）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| code | VARCHAR(50) | 车间编码，唯一 |
| name | VARCHAR(100) | 车间名称 |
| calendar_id | UUID FK (nullable) | 关联工厂日历 |
| manager_name | VARCHAR(50) | 负责人 |
| enabled | BOOLEAN | 是否启用 |
| sort_order | INT | 排序 |
| description | VARCHAR(500) | 备注 |
| create_time / update_time | TIMESTAMP | 时间戳 |

### 2. 改造 Resource（注塑机增强）

在已有 `resources` 表上新增字段：

| 新增字段 | 类型 | 说明 |
|---------|------|------|
| workshop_id | UUID FK (nullable) | 所属车间 |
| tonnage | INT | 锁模力（吨），注塑机核心参数 |
| machine_brand | VARCHAR(50) | 机台品牌 |
| machine_model | VARCHAR(50) | 机台型号 |
| max_shot_weight | DECIMAL(10,2) | 最大射出量（g） |
| status | VARCHAR(20) | 机台状态（RUNNING/IDLE/MAINTENANCE/DISABLED） |
| calendar_id | UUID FK (nullable) | 设备级日历（不设则继承车间） |

> 所有新字段 nullable，兼容已有数据。

### 日历继承（两级）

```
Resource.calendar_id ?? Workshop.calendar_id ?? 全局默认日历
```

### 枚举

```java
enum MachineStatus { RUNNING, IDLE, MAINTENANCE, DISABLED }
```

## 实施步骤

### Step 1：后端数据库 & 实体

1. **Flyway 迁移** `V10__Workshop_And_Machine.sql`
   - 创建 `workshops` 表
   - `resources` 表新增 `workshop_id`、`tonnage`、`machine_brand`、`machine_model`、`max_shot_weight`、`status`、`calendar_id` 列
   - 外键 & 索引
   - 权限种子数据

2. **Domain 实体** (`aps-domain`)
   - 新增 `Workshop.java`
   - 修改 `Resource.java`，增加注塑机属性
   - 新增 `MachineStatus.java` 枚举

3. **Repository** (`aps-service`)
   - `WorkshopRepository.java`
   - 修改 `ResourceRepository.java`（按车间/吨位查询）

4. **Service** (`aps-service`)
   - `WorkshopService.java` — 车间 CRUD + 获取有效日历
   - 修改 `Resource` 相关逻辑，支持按车间筛选

5. **Controller & DTO** (`aps-api`)
   - `WorkshopController.java`
   - `WorkshopDto.java`
   - 修改 `ResourceDto`/`ResourceController` 支持新字段

### Step 2：前端页面

6. **API** — `aps-web/src/api/workshop.ts`
7. **类型** — 补充 `Workshop`、增强 `Resource` 接口
8. **页面** — `WorkshopManagement.vue`
   - 上方：车间选择（Tab 或卡片），显示车间信息 + 关联日历
   - 下方：该车间的注塑机列表（表格），显示吨位、品牌、状态等
   - 支持新增/编辑/删除车间和注塑机
9. **路由** — `/workshop` 路由，group: `基础数据`

## API 设计

| Method | Path | 说明 |
|--------|------|------|
| GET | `/api/workshops` | 车间列表 |
| POST | `/api/workshops` | 创建车间 |
| PUT | `/api/workshops/{id}` | 更新车间 |
| DELETE | `/api/workshops/{id}` | 删除车间 |
| GET | `/api/workshops/{id}/resources` | 获取车间下注塑机列表 |
| GET | `/api/workshops/{id}/effective-calendar` | 获取车间的有效日历 |
| GET | `/api/resources` | 资源列表（支持 workshopId/tonnage 筛选） |
| POST | `/api/resources` | 创建资源（注塑机） |
| PUT | `/api/resources/{id}` | 更新资源 |
| DELETE | `/api/resources/{id}` | 删除资源 |
