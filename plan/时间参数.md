# Role
你是一名资深的 Java/Spring Boot 专家，擅长工业级 APS（高级计划与排程）系统建模。

# Context
我正在开发一个 APS 系统。现在需要实现“排程时间参数”配置功能。
这些参数定义了排程引擎（基于 Timefold Solver）在处理特定工作中心（Work Center）时的行为边界。

# Domain Logic (Core Requirements)
请根据以下逻辑编写/重构代码：
1. **排程视窗过滤**：
   - 必须根据 `scheduling_start_days`（排程起点天数）和 `scheduling_horizon_days`（排程天数）来过滤待排产的工单工序（Task/Operation）。
   - 计算公式：`Start_Date = Today + scheduling_start_days`, `End_Date = Start_Date + scheduling_horizon_days`。
2. **冻结期保护（Frozen Zone）**：
   - 凡是落在 `Today` 到 `Start_Date` 之间的既有计划，必须标记为 `pinned`（固定），排程引擎不可移动。
3. **多设备支持**：
   - 系统支持针对不同的 `resources` 设置不同的时间参数，逻辑需支持按 resources ID 索引配置。
4. **技术要求**：
   - 数据库使用 PostgreSQL，主键使用 UUID。
   - 实体类需符合 JPA 规范。
   - 提供一个 Service 方法：`getPlanningWindow(UUID workCenterId)`，返回计算后的开始与结束时间。

# Task
1. 请生成对应的实体类 `ScheduleTimeParameter`。
2. 编写核心逻辑 Service，用于将数据库中的配置转换为排程引擎可用的 `PlanningValueRange` 或时间过滤谓词。
3. 考虑到性能，请使用 Java 8+ Time API（LocalDateTime）。
