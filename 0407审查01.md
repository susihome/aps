# 🔴 致命级（会导致启动失败或核心功能不可用）

## 1. Flyway 迁移脚本冲突 — 启动必崩

`@d:\claudework\aps\aps-api\src\main\resources\db\migration\V1__Initial_Schema.sql` 已创建 `users`、`roles`、`permissions`、`user_roles`、`role_permissions`、`audit_logs` 等表。

`@d:\claudework\aps\aps-api\src\main\resources\db\migration\V002__create_auth_tables.sql` 再次用 `CREATE TABLE` 创建**完全相同的表**（且列定义不同：`BIGSERIAL` vs `UUID`、`password_hash` vs `password`）。

**结果**：Flyway 按版本号顺序执行（V1 → V002），V002 会因表已存在而直接失败，**应用无法启动**。

## 2. DB Schema 与 JPA Entity 列名不匹配 — `ddl-auto: validate` 必崩

当前 [application.yml](cci:7://file:///d:/claudework/aps/aps-api/target/classes/application.yml:0:0-0:0) 配置的是 `ddl-auto: validate`，Hibernate 启动时会严格校验表结构。但 V1 脚本的列名和实体映射**至少有 3 处不匹配**：

| SQL 列名 (V1) | JPA 属性 | Hibernate 期望列名 | 结果 |
|---|---|---|---|
| `users.password` | `User.passwordHash` | `password_hash` | ❌ 校验失败 |
| `permissions.name` | `Permission.code` | `code` | ❌ 校验失败 |
| `schedules.score` | `Schedule.finalScore` | `final_score` | ❌ 校验失败 |

## 3. CSRF 阻断所有认证后的写操作

```@d:\claudework\aps\aps-api\src\main\java\com\aps\api\config\SecurityConfig.java:39-41
.csrf(csrf -> csrf
    .ignoringRequestMatchers("/api/auth/**", "/ws/**")
)
```

只有 `/api/auth/**` 和 `/ws/**` 豁免 CSRF。但会话策略是 `STATELESS`（无 Session），CSRF token 基于 Session 存储，**所有认证后的 POST/PUT/DELETE 请求都会被 CSRF 拦截返回 403**。

对于 cookie-based JWT 架构，通常要么全局禁用 CSRF（配合 `SameSite` cookie），要么用 `CookieCsrfTokenRepository`。当前两者都没做。

## 4. Solver `endTime` 永远不会被重算 — 约束全部失效


`startTime` 是 `@PlanningVariable`，Solver 会修改它。但 `endTime` **不是** `@ShadowVariable`，也没有任何 listener 根据 `startTime + duration` 重算 `endTime`。

**结果**：Solver 改了 `startTime` 后，`endTime` 仍是旧值。所有依赖 `endTime` 的约束（[resourceConflict](cci:1://file:///d:/claudework/aps/aps-solver/src/main/java/com/aps/solver/ApsConstraintProvider.java:22:4-39:5)、[operationSequence](cci:1://file:///d:/claudework/aps/aps-solver/src/main/java/com/aps/solver/ApsConstraintProvider.java:41:4-63:5)、[minimizeDelay](cci:1://file:///d:/claudework/aps/aps-solver/src/main/java/com/aps/solver/ApsConstraintProvider.java:65:4-79:5)）**全部在用过期数据评估**，求解结果无意义。
