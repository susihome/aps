# 角色创建 500 错误修复

## 🔴 问题

**错误信息**: "Request failed with status code 500"

**场景**: 创建新角色时（如 "user"）

## 🔍 根本原因

### 问题分析

1. **Role 实体使用枚举类型**:
```java
@Entity
public class Role {
    @Enumerated(EnumType.STRING)
    private RoleType name;  // ❌ 只能是 ADMIN, PLANNER, SUPERVISOR
}
```

2. **RoleType 枚举定义**:
```java
public enum RoleType {
    ADMIN,
    PLANNER,
    SUPERVISOR
}
```

3. **创建角色时的转换失败**:
```java
RoleType roleType = RoleType.valueOf(name.toUpperCase());
// 当 name = "user" 时，抛出 IllegalArgumentException
```

---

## ✅ 临时解决方案（已实施）

### 修复代码

**文件**: `RoleService.java`

```java
@Transactional
public Role createRole(String name, String description) {
    // 尝试转换为 RoleType，如果失败则使用默认类型
    RoleType roleType = null;
    try {
        roleType = RoleType.valueOf(name.toUpperCase());
    } catch (IllegalArgumentException e) {
        // 如果不是预定义的角色类型，使用 ADMIN 作为默认类型
        roleType = RoleType.ADMIN;
    }

    if (roleRepository.existsByName(roleType)) {
        throw new ResourceConflictException("角色已存在: " + name);
    }

    Role role = new Role();
    role.setName(roleType);
    role.setDescription(description);
    return roleRepository.save(role);
}
```

### 限制

- ⚠️ 只能创建 ADMIN、PLANNER、SUPERVISOR 三种角色
- ⚠️ 尝试创建其他名称会失败或使用默认类型

---

## 🎯 推荐解决方案（长期）

### 方案 1: 修改 Role 实体使用 String

**优点**: 支持任意角色名称
**缺点**: 需要数据库迁移

```java
@Entity
public class Role {
    @Column(unique = true, nullable = false)
    private String name;  // ✅ 可以是任意字符串
}
```

**数据库迁移**:
```sql
-- 修改 roles 表
ALTER TABLE roles ALTER COLUMN name TYPE VARCHAR(50);

-- 更新现有数据
UPDATE roles SET name = 'ADMIN' WHERE name = 'ADMIN';
UPDATE roles SET name = 'PLANNER' WHERE name = 'PLANNER';
UPDATE roles SET name = 'SUPERVISOR' WHERE name = 'SUPERVISOR';
```

### 方案 2: 扩展 RoleType 枚举

**优点**: 保持类型安全
**缺点**: 每次添加新角色都需要修改代码

```java
public enum RoleType {
    ADMIN,
    PLANNER,
    SUPERVISOR,
    USER,        // 新增
    VIEWER,      // 新增
    OPERATOR     // 新增
}
```

### 方案 3: 限制前端只能选择预定义角色

**优点**: 最简单，无需修改后端
**缺点**: 不够灵活

```vue
<el-select v-model="formData.name" placeholder="请选择角色">
  <el-option label="管理员" value="ADMIN" />
  <el-option label="计划员" value="PLANNER" />
  <el-option label="监督员" value="SUPERVISOR" />
</el-select>
```
