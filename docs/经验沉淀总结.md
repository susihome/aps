## 已保存的经验清单

### 1. Spring Security 授权表达式与自定义 Principal
**文件：** `memory/feedback_spring_security_principal.md`

**核心教训：**
- `@PreAuthorize` 表达式中访问的字段必须在 Principal 对象中实际存在
- 不能假设 `authentication.principal.id` 可用，需要自定义 `UserPrincipal`

**关键代码：**
```java
public class UserPrincipal extends User {
    private final UUID id;
}
```

---

### 2. Timefold Solver 模型分离架构
**文件：** `memory/feedback_timefold_model_separation.md`

**核心教训：**
- 不要让同一个类同时承担 `@Entity` 和 `@PlanningEntity` 职责
- JPA 和 Timefold 生命周期冲突，必须分离

**关键架构：**
```
JPA Entity (Assignment)
    ↓ 深拷贝转换
Planning Model (AssignmentPlanningModel)
    ↓ ID 映射回写
JPA Entity (保存结果)
```

---

### 3. Timefold 约束规则空值保护
**文件：** `memory/feedback_timefold_null_safety.md`

**核心教训：**
- Planning variables 在初始化阶段允许为 `null`
- 所有约束规则必须过滤空值，避免 NPE

**关键模式：**
```java
.filter(a -> a.getAssignedResource() != null
        && a.getStartTime() != null
        && a.getEndTime() != null)
```

---

### 4. API 返回格式统一
**文件：** `memory/feedback_api_response_format.md`

**核心教训：**
- 成功响应和失败响应必须使用相同格式
- 避免前端需要按接口分支解析

**统一格式：**
```java
public AjaxResult<T> {
    private int code;
    private String message;
    private T data;
}
```

---

### 5. 安全配置不要硬编码
**文件：** `memory/feedback_security_config_externalization.md`

**核心教训：**
- Cookie `secure`、CORS、HTTPS 等安全配置必须配置化
- 避免硬编码导致生产环境风险

**配置驱动：**
```yaml
jwt:
  cookie:
    secure: ${JWT_COOKIE_SECURE:false}
```

---

### 6. AOP 审计链路完整性
**文件：** `memory/feedback_audit_userid_extraction.md`

**核心教训：**
- 审计日志必须记录 `userId`，不能只记录 `username`
- 从 `UserPrincipal` 提取完整用户信息

**提取逻辑：**
```java
if (authentication.getPrincipal() instanceof UserPrincipal) {
    UserPrincipal principal = (UserPrincipal) authentication.getPrincipal();
    userId = principal.getId();
}
```

---

### 7. 浅拷贝 vs 深拷贝陷阱
**文件：** `memory/feedback_deep_copy_vs_shallow_copy.md`

**核心教训：**
- `new ArrayList<>(list)` 只复制容器，元素还是同一批引用
- 必须为每个元素创建新对象才是真正的深拷贝

**深拷贝模式：**
```java
for (Assignment assignment : schedule.getAssignments()) {
    AssignmentPlanningModel planningAssignment = new AssignmentPlanningModel(
        assignment.getId(),
        assignment.getOperation()
    );
    // 复制字段...
    planningAssignments.add(planningAssignment);
}
```
