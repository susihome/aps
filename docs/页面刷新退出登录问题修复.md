# 页面刷新退出登录问题修复

## 🔴 问题

**症状**: 每次刷新页面就退到登录页面

## 🔍 根本原因

### 问题分析

1. **页面刷新时 Pinia store 被重置**
   - `user.value` 变为 `null`
   - `isAuthenticated` 返回 `false`

2. **路由守卫执行时机问题**
   - 路由守卫在 `router.beforeEach` 中立即检查认证状态
   - 但此时 `authStore.initialize()` 可能还未完成
   - 导致认证检查失败，重定向到登录页

3. **初始化竞态条件**
   ```typescript
   // main.ts 中的初始化
   authStore.initialize().finally(() => {
     app.mount('#app')
   })
   
   // 但路由守卫可能在初始化完成前就执行了
   router.beforeEach(async (to, from, next) => {
     if (!authStore.isAuthenticated) {  // ❌ 此时可能还未初始化完成
       next({ name: 'Login' })
     }
   })
   ```

---

## ✅ 解决方案

### 修复方法：在路由守卫中等待初始化

**文件**: `router/index.ts`

```typescript
// 标记初始化是否完成
let isInitialized = false

// 路由守卫
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()

  // 首次进入时，等待初始化完成
  if (!isInitialized) {
    try {
      await authStore.initialize()
      isInitialized = true
    } catch (error) {
      console.error('初始化认证失败:', error)
      isInitialized = true
    }
  }

  // 后续的认证检查...
  if (to.meta.requiresAuth) {
    if (!authStore.isAuthenticated) {
      next({ name: 'Login', query: { redirect: to.fullPath } })
      return
    }
  }

  next()
})
```

### 工作原理

1. **首次路由导航时**:
   - 检查 `isInitialized` 标志
   - 如果未初始化，调用 `authStore.initialize()`
   - 等待初始化完成后再进行认证检查

2. **后续路由导航时**:
   - `isInitialized` 为 `true`
   - 跳过初始化，直接进行认证检查

3. **初始化过程**:
   - 调用 `/auth/me` API 获取当前用户信息
   - 如果成功，设置 `user.value`
   - 如果失败（如 token 过期），`user.value` 保持 `null`

## 🔒 认证流程

### 完整的认证流程

1. **用户登录**:
   ```typescript
   authApi.login(credentials)
     → 后端返回 token (存储在 cookie 中)
     → 设置 user.value
   ```

2. **页面刷新**:
   ```typescript
   路由守卫触发
     → 检查 isInitialized
     → 调用 authStore.initialize()
     → 调用 authApi.getCurrentUser()
     → 后端验证 cookie 中的 token
     → 返回用户信息
     → 设置 user.value
   ```

3. **后续导航**:
   ```typescript
   路由守卫触发
     → isInitialized = true
     → 直接检查 authStore.isAuthenticated
     → 允许/拒绝导航
   ```

## 🎯 其他注意事项

### Token 存储

确保后端使用 **HttpOnly Cookie** 存储 token：
```java
Cookie cookie = new Cookie("token", token);
cookie.setHttpOnly(true);  // 防止 XSS
cookie.setSecure(true);    // 仅 HTTPS
cookie.setPath("/");
cookie.setMaxAge(7 * 24 * 60 * 60);  // 7 天
response.addCookie(cookie);
```

### Axios 配置

确保 axios 发送 cookie：
```typescript
const axiosInstance = axios.create({
  baseURL: '/api',
  withCredentials: true,  // ✅ 重要：发送 cookie
  timeout: 30000
})
```

### CORS 配置

后端需要允许携带凭证：
```java
@Configuration
public class CorsConfig {
  @Bean
  public CorsFilter corsFilter() {
    CorsConfiguration config = new CorsConfiguration();
    config.setAllowCredentials(true);  // ✅ 允许凭证
    config.addAllowedOrigin("http://localhost:5173");
    config.addAllowedHeader("*");
    config.addAllowedMethod("*");
    // ...
  }
}
```

## 🐛 故障排查

如果问题仍然存在，检查：

1. **浏览器控制台**:
   - 查看 `/auth/me` API 是否返回 401
   - 查看是否有 CORS 错误

2. **Network 标签**:
   - 检查请求是否携带 cookie
   - 检查响应状态码

3. **Application 标签**:
   - 检查 cookie 是否存在
   - 检查 cookie 的 domain 和 path

4. **后端日志**:
   - 检查 token 验证是否失败
   - 检查是否有异常抛出
