```java
    @NotNull(message = "权限类型不能为空")

    @NotBlank(message = "图标不能为空")
    private String icon;

    @NotNull(message = "排序号不能为空")
    private Integer sort;
    // ... 其他字段
}
```

**控制器更新**：
```java
@PostMapping

    // ✅ @Valid 触发验证
```

### 3. 过度宽松的 CORS 配置
**问题**：`setAllowedHeaders(Arrays.asList("*"))` 允许任意请求头
- 启用基于头的攻击
- 凭证泄露风险

**修复**：白名单特定请求头
```java
configuration.setAllowedHeaders(Arrays.asList(
    "Content-Type",
    "Authorization",
    "Accept",
    "X-Requested-With",
    "X-CSRF-Token"
));
configuration.setExposedHeaders(Arrays.asList("Authorization", "X-Total-Count"));
```

### 4. N+1 查询问题（批量排序）
**问题**：`updateSort()` 为每个权限执行一次查询
- 100 个权限 = 100+ 次数据库往返
- 性能严重下降

**修复**：实现批量更新
```java
// PermissionRepository.java
@Modifying
@Query("UPDATE Permission p SET p.sort = CASE WHEN p.id = :id THEN :sort ELSE p.sort END WHERE p.id IN :ids")
void updateSortBatch(@Param("ids") List<UUID> ids, @Param("id") UUID id, @Param("sort") Integer sort);

// PermissionService.java
@Transactional
public void updateSort(List<SortUpdate> updates) {
    if (updates == null || updates.isEmpty()) return;
    
    for (SortUpdate update : updates) {
        permissionRepository.updateSortBatch(
            updates.stream().map(SortUpdate::getId).toList(),
            update.getId(),
            update.getSort()
        );
    }
    log.info("权限排序已更新: {} 条记录", updates.size());
}
```

### 5. 循环引用未防止
**问题**：权限可以设置为其自身或子权限的父权限
- 破坏树形结构
- 导致查询失败

**修复**：添加循环检测
```java
@Transactional
public Permission updatePermission(UUID id, Permission updates) {
    Permission permission = getPermission(id);
    
    // ... 其他更新 ...
    
    // 更新父权限（检查循环引用）
    if (updates.getParent() != null && updates.getParent().getId() != null) {
        UUID parentId = updates.getParent().getId();
        
        // 防止自引用
        if (parentId.equals(id)) {
            throw new ResourceConflictException("不能将权限设置为其自身的父权限");
        }
        
        // 防止设置为子权限
        if (isAncestor(id, parentId)) {
            throw new ResourceConflictException("不能将权限设置为其子权限的父权限");
        }
        
        Permission parent = getPermission(parentId);
        permission.setParent(parent);
    }
    
    return permissionRepository.save(permission);
}

private boolean isAncestor(UUID permissionId, UUID potentialAncestorId) {
    Permission current = permissionRepository.findById(potentialAncestorId).orElse(null);
    while (current != null) {
        if (current.getId().equals(permissionId)) {
            return true;
        }
        current = current.getParent();
    }
    return false;
}
```

- 检查 `error.response?.status !== 'cancel'` 错误

**修复**：

```typescript
async function handleDeletePermission(id: string) {

    }
  }
}
```

## 🟡 MEDIUM 优先级优化建议

### 数据库优化

#### 1. N+1 查询问题（权限树）
**当前**：`findPermissionTree()` 只获取 2 级
- 3 级权限结构会触发 N+1 查询

**建议**：使用递归 CTE 或在服务层构建树
```sql
-- PostgreSQL 递归 CTE
@Query(value = """
    WITH RECURSIVE permission_tree AS (
        SELECT id, code, name, type, parent_id, sort, enabled, visible, 0 as level
        FROM permissions
        WHERE parent_id IS NULL
        UNION ALL
        SELECT p.id, p.code, p.name, p.type, p.parent_id, p.sort, p.enabled, p.visible, pt.level + 1
        FROM permissions p
        INNER JOIN permission_tree pt ON p.parent_id = pt.id
    )
    SELECT * FROM permission_tree ORDER BY level, sort
    """, nativeQuery = true)
List<Permission> findPermissionTreeRecursive();
```

#### 2. 索引优化
**当前**：分离的索引
```sql
CREATE INDEX idx_permission_parent_id ON permissions(parent_id);
CREATE INDEX idx_permission_sort ON permissions(sort);
```

**优化**：复合索引 + 部分索引
```sql
-- 复合索引用于常见查询模式
CREATE INDEX idx_permission_parent_sort ON permissions(parent_id, sort) WHERE parent_id IS NOT NULL;
CREATE INDEX idx_permission_root_sort ON permissions(sort) WHERE parent_id IS NULL;

-- 部分索引用于启用状态查询
CREATE INDEX idx_permission_enabled_true ON permissions(enabled) WHERE enabled = true;

-- 显式索引用于编码查询
CREATE UNIQUE INDEX idx_permission_code ON permissions(code);
```

#### 3. 缓存策略
**建议**：权限树很少变化，应该缓存
```java
@Cacheable(value = "permission:tree", unless = "#result == null")
@Transactional(readOnly = true)
public List<Permission> getPermissionTree() {
    return permissionRepository.findPermissionTree();
}

// 更新时清除缓存
@CacheEvict(value = "permission:tree", allEntries = true)
@Transactional
public Permission createPermission(Permission permission) { ... }

@CacheEvict(value = "permission:tree", allEntries = true)
@Transactional
public Permission updatePermission(UUID id, Permission updates) { ... }

@CacheEvict(value = "permission:tree", allEntries = true)
@Transactional
public void deletePermission(UUID id) { ... }
```
